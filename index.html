Kim and Bey's git workflow

  • Each of them has a local copy of the book which they can save as they work on it without it being automatically propagated to the server
  • When they're happy with a change they've done and want to save this state, they commit their changes on their own branch
  • They push this up to github and make a pull request to master. They review each others' changes and merge them
